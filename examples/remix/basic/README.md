# `@tanstack/remix-router` + `@tanstack/remix-start` example

End-to-end demo of TanStack Router on Remix 3 ‚Äî the routing layer
rendered through `@remix-run/ui`, the full-stack runtime via
`@tanstack/remix-start`, the production-bundle compiler via
`@tanstack/remix-start/plugin/vite`.

## Routes

| Path | Demonstrates |
|---|---|
| `/` | Static home page |
| `/users` | Loader-driven list, `<Link>` to nested detail |
| `/users/$id` | `useLoaderData` + `useParams`, server-fn-rendered HTML mounted via `innerHTML` |
| `/posts` | Same shape with markdown content |
| `/posts/$slug` | Heavy markdown + syntax highlighting (server-only deps stay out of client bundle) |
| `/admin/users/$userId/sessions/$sessionId` | 4-deep nested layout; exercises `<Outlet>`/`<MatchContext>` reactivity at every level |
| `/catalog` | `validateSearch`, `loaderDeps`, `<Link search={updater}>`, form-driven `useNavigate` |
| `/slow` | Async loader (800ms), `pendingComponent` UI |
| `/deferred` | `defer()` + `<Await>` ‚Äî sync part renders immediately, slow chunk streams in |
| `/lab/error` | Loader throws ‚Üí `errorComponent` |
| `/lab/missing` | Loader calls `notFound()` ‚Üí `notFoundComponent` |
| `/lab/render-error` | Render-time throw caught by enclosing `<CatchBoundary>` |
| `/guestbook` | `createServerFn({ method: 'POST' })` with `inputValidator`; form submit calls server fn from event handler |
| `/counter` | `clientEntry()`-marked island that hydrates standalone (counter `+`/`reset` buttons) |

## Bundle savings

The Vite plugin (`@tanstack/remix-start/plugin/vite` ‚Üí
`@tanstack/start-plugin-core` under the hood) extracts `createServerFn`
handler bodies from the client bundle, replacing them with RPC
fetchers that hit `${TSS_SERVER_FN_BASE}/<id>`. Heavy server-only
modules (markdown renderers, ORM clients, image pipelines) reachable
only through those handlers fall out of the client bundle entirely.

In this example: `marked`, `highlight.js`, language grammars, and
`heavyDep` (a deliberately-fat user-bio renderer) are all server-only
‚Äî the client bundle (~190 KB minified) contains none of them.

### Client hydration

`__root.tsx` renders only the body content (`<nav>`, `<Outlet>`).
`<StartServer>` (server-only) provides the shell. On hydration:
`createRoot(document.body)` matches the body-content tree against
existing nodes ‚Äî no `<html>`-as-render-root mismatch.

### Reactivity

The Remix UI runtime updates components by re-running the render
function on `handle.update()`. The binding's `subscribeStore` and
`subscribeDynamicStore` (the latter for prop-derived store identities
like the active matchId) wire router stores to handle updates.
`MatchContext` and `RouterContextProvider` set their context values in
the render function (not setup) so prop changes propagate through
reused vnode instances.
