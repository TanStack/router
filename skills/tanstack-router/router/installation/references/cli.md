# Router CLI Installation

Using TanStack Router CLI for route generation without bundler plugins.

## When to Use CLI

Use the standalone CLI when:

- Your bundler isn't supported by the plugin
- You want more control over when route generation runs
- You're using a custom build setup

## Installation

```bash
npm install -D @tanstack/router-cli
```

## Commands

### Generate Routes (One-time)

```bash
npx tsr generate
```

### Watch Mode (Development)

```bash
npx tsr watch
```

## Package.json Scripts

```json
{
  "scripts": {
    "dev": "tsr watch & vite",
    "build": "tsr generate && vite build",
    "generate-routes": "tsr generate"
  }
}
```

## Configuration File

Create `tsr.config.json` in project root:

```json
{
  "routesDirectory": "./src/routes",
  "generatedRouteTree": "./src/routeTree.gen.ts",
  "routeFileIgnorePrefix": "-",
  "quoteStyle": "single"
}
```

## Configuration Options

| Option                  | Default                  | Description                          |
| ----------------------- | ------------------------ | ------------------------------------ |
| `routesDirectory`       | `./src/routes`           | Directory containing route files     |
| `generatedRouteTree`    | `./src/routeTree.gen.ts` | Output path for generated route tree |
| `routeFileIgnorePrefix` | `-`                      | Prefix for ignored files             |
| `quoteStyle`            | `single`                 | Quote style in generated code        |
| `semicolons`            | `true`                   | Include semicolons in generated code |

## Project Structure

```
project/
├── tsr.config.json        # CLI configuration
├── src/
│   ├── routes/
│   │   ├── __root.tsx
│   │   ├── index.tsx
│   │   └── about.tsx
│   ├── routeTree.gen.ts   # Generated by CLI
│   └── main.tsx
```

## Combining with Bundler

If you want to use both CLI and bundler plugin (e.g., for CI):

```json
{
  "scripts": {
    "dev": "vite",
    "build": "tsr generate && vite build",
    "generate": "tsr generate"
  }
}
```

The bundler plugin handles dev mode, CLI handles production builds.
